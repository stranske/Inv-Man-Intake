# app-baseline-kit dependency

Inv-Man-Intake **vendors** the baseline harness rather than depending on the git package. The initial commit (ae5784e) declared `app-baseline-kit @ git+...@main` in `[project.optional-dependencies].dev`, but within the same PR commit 35d3714 **removed the git reference and vendored the entire module** into `src/baseline_kit/__init__.py` (a ~99-line pure-Python harness whose only dependency, PyYAML, is already provided by pytest-regressions). Vendoring was chosen to avoid lock-file churn from an `@main` pin tracking an external repo's frequent updates — it was not a backend limitation (setuptools serializes URL-in-extra metadata fine) nor a uv conflict. `baseline_kit` is declared first-party in `.project_modules.txt` so dependency-sync scripts recognize it; tests import the local copy directly. The copy is intentionally decoupled — if upstream baseline_kit changes, sync manually. No item-2 change needed.
